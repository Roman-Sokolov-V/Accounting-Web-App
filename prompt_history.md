⚙️ Еволюція розробки: Від логіки до архітектури

    UI & UX: Реалізація адаптивної таблиці даних у Streamlit з використанням "зважених" колонок (use_container_width) для оптимального відображення фінансових реєстрів.

    Database Queries: Перехід від базових фільтрів до використання filter_by та складних умов через and_/or_ для точного вибору бухгалтерських проводок.

    Data Aggregation: Впровадження функцій func.sum та coalesce для розрахунку фінансових сальдо, враховуючи специфіку None значень у базах даних.

    Accounting Logic: Побудова надійної системи подвійного запису (Double-entry bookkeeping). Коректна реалізація дебету/кредиту для активних та пасивних рахунків (Asset, Liability, Equity, Revenue, Expense).

    Transaction Integrity: Проектування атомарних операцій: поділ нарахування витрат (EXPENSE) та фактичної оплати (PAYMENT_SENT) для забезпечення точності обліку заборгованості (Accounts Payable/Receivable).

    ORM Optimization: Вирішення проблеми N+1 за допомогою joinedload (Eager Loading) для миттєвого отримання транзакцій разом із пов’язаними записами.

    Performance & Audit: Впровадження бізнес-дати та системного часу (created_at). Денормалізація дати в таблицю проводок для прискорення звітів.

    Indexing Strategy: Створення складених індексів (Composite Index) на (account_id, date), що забезпечує миттєву генерацію балансових звітів навіть на великих обсягах даних.
